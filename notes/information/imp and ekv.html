Задание 2: Логика и Таблицы

Приоритет операций

1. Скобки и NOT not x
2. Конъюнкция (И) x and y
3. Дизъюнкция (ИЛИ) x or y
4. Импликация x <= y
5. Эквиваленция x == y

* Если сомневаешься в приоритете — всегда ставь лишние скобки!

Методика решения (Алгоритм)

1 Генерация всех комбинаций: Используем вложенные циклы для x, y, z, w. Каждая переменная принимает значения 0 и 1.
2 Фильтрация по значению F: В условии задачи всегда сказано, чему равна функция (0 или 1). Выводим только нужные строки. 
print('x y z w | F')
for x in 0,1:
    for y in 0,1:
        for z in 0,1:
            for w in 0,1:
                # Записываем формулу в одну строку
                f = (x <= y) and (y <= z) and (z <= w)
                
                # Допустим, в таблице из КИМа везде стоят нули (F=0)
                if f == 0:
                    print(x, y, z, w, '|', int(f))
3 Анализ «уникальных» признаков: Получив таблицу из Python (обычно там 3-4 строки), сравниваем её с фрагментом из задания.

На что смотреть при сопоставлении:

  • Вес столбца: Посчитай количество единиц в каждом столбце твоей таблицы. Если в одном столбце три "1", а во всех других меньше — ищи такой же столбец в задании.
  • Уникальные строки: Ищи строку, где только одна единица (или только один ноль). Она станет "зацепкой".
  • Пустые ячейки: Не пытайся их заполнить сразу. Сначала расставь те переменные, в которых ты уверен на 100%.

Pro-Tips

1. Итераторы: Вместо 4-х циклов можно использовать itertools.product(range(2), repeat=4).

2. Отрицание: В Python not имеет очень высокий приоритет. Конструкция not x == y превратится в (not x) == y. Если нужно отрицать всё равенство, пиши not (x == y).
